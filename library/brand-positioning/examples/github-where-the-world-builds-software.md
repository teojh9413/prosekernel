# GitHub where the world builds software

## Source
- Author: GitHub
- URL: https://github.com/about
- Rights note: Metadata and original craft analysis only; do not mirror full source text.

## Why this is good
GitHub’s positioning works by making the product sound less like a tool and more like the location of modern software collaboration. It leverages community scale as proof and identity as pull.

## Craft moves
- Positions around place and community, not feature inventory.
- Turns scale into defaultness.
- Makes the user feel part of a professional tribe.
- Keeps the category broad enough for many workflows.

## Structure map
1. Place/identity claim.
2. Community proof.
3. Core workflows.
4. Invitation to participate.

## Reusable pattern
For platform brands, position the product as the place where a valued community does its defining work.

## Imitation prompt
Write platform positioning that turns the product into a credible place/community, then supports it with workflows and proof.

## Anti-patterns to avoid
- Listing platform features with no identity.
- Claiming community without evidence.
- Over-narrowing a multi-use platform.
