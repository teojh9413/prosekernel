# Okta — January 2022 Security Incident FAQ

## Why this is good
The useful craft move is scope control. Security incidents create information hunger; the writer must answer what is known, what is not known, who is affected, and what customers should do without overclaiming certainty.

## Craft moves
- Uses update/FAQ structure to meet customer questions directly.
- Clarifies scope and affected systems.
- Distinguishes evidence from inference.
- Gives customer-facing action guidance.
- Keeps the communication anchored in facts available at the time.

## Structure map
1. State the latest update.
2. Define incident scope.
3. Answer key customer questions.
4. Explain recommended actions.
5. Commit to further updates as facts change.

## Excerpt or summary
Summary: Okta’s incident update is a useful model for security crisis writing that needs to manage uncertainty while still being practically helpful.

## Reusable pattern
For security FAQ updates: “Known: [facts]. Scope: [affected/unaffected]. Customer action: [steps]. Unknown or under review: [items]. Next update: [channel/timing].”

## Imitation prompt
Write a security incident FAQ with Okta-style scope discipline. Answer the questions customers actually have and mark unknowns clearly.

## Anti-patterns to avoid
- Over-certainty before investigation is complete.
- Burying customer action inside narrative.
- Letting legal caution erase practical usefulness.
