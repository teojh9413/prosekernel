# Apple HIG writing guidance

## Source
- Author: Apple
- URL: https://developer.apple.com/design/human-interface-guidelines/writing
- Rights note: Metadata and original craft analysis only; do not mirror full source text.

## Why this is good
Apple’s writing guidance matters because it links copy quality to interaction quality. Short text is not automatically good; it is good when it reduces hesitation and names the right action at the right time.

## Craft moves
- Prioritizes clarity over brand flourish.
- Uses familiar user language.
- Keeps interface text close to action.
- Treats brevity as a usability constraint.

## Structure map
1. User task.
2. Plain instruction or label.
3. Only necessary context.
4. Consistent terminology.

## Excerpt or summary
Design guideline analysis only; no source text mirrored.

## Reusable pattern
Good UI writing removes the smallest possible amount of uncertainty at the point of action.

## Imitation prompt
Rewrite this UI flow with brief, familiar, action-adjacent copy that avoids brand flourish.

## Anti-patterns to avoid
- Brevity that omits necessary context.
- Marketing language inside task flows.
- Synonyms that break consistency.
