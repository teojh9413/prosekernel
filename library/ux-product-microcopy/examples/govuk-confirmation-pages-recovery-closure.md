# GOVUK confirmation pages recovery closure

## Source
- Author: GOV.UK Design System
- URL: https://design-system.service.gov.uk/patterns/confirmation-pages/
- Rights note: Open-license source; ProseKernel stores original analysis and uses the source URL for provenance.

## Why this is good
GOV.UK confirmation guidance is strong because it handles the moment after action. It does not stop at “success”; it tells the user what has happened, what evidence they have, and what to expect next.

## Craft moves
- Confirms completion in plain language.
- Provides reference or evidence when needed.
- Names next steps and timing.
- Reduces post-submit anxiety.

## Structure map
1. Clear success state.
2. Reference/detail if relevant.
3. What happens next.
4. What the user should do if something is wrong.

## Excerpt or summary

## Reusable pattern
After important user action, confirmation copy should close the loop: done, evidence, next step, recovery path.

## Imitation prompt
Write a confirmation page for a completed service action with success, reference detail, next timing, and recovery guidance.

## Anti-patterns to avoid
- Celebratory success messages that omit next steps.
- Vague “we’ll be in touch” copy.
- Making users wonder whether the action actually completed.
