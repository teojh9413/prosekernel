# GOV.UK — Start Pages Task Entry

## Source
- Author: GOV.UK Service Manual
- URL: https://www.gov.uk/service-manual/design/start-pages
- Rights note: Metadata and original analysis only. Link to the source instead of copying it wholesale.

## Why this is good
GOV.UK start page guidance is effective because it respects task anxiety. The page is not a marketing landing page; it is a threshold into a service. Good microcopy reduces uncertainty before the user commits.

## Craft moves
- Names the user task plainly.
- Sets expectations about eligibility, documents, or time needed.
- Uses the page to prevent avoidable failure later in the flow.
- Separates service entry from persuasive marketing copy.
- Makes the next action obvious.

## Structure map
1. Name the service or task.
2. State who can use it and any preconditions.
3. List what the user needs before starting.
4. Explain what happens after they proceed.
5. Provide a clear start action.

## Excerpt or summary
Summary: GOV.UK start pages show microcopy as preparation. The best entry copy helps users decide whether to begin and prevents later friction.

## Reusable pattern
For a service start page: “Use this service to [task]. Before you start, you’ll need [inputs]. It takes about [time/effort]. After you submit, [next step]. Start now.”

## Imitation prompt
Write a service/onboarding entry screen using GOV.UK start-page logic. Be plain, preparatory, and action-oriented. Do not use hype or vague reassurance.

## Anti-patterns to avoid
- Selling the service instead of preparing the user.
- Hiding requirements until late in the flow.
- Using clever button copy when the user needs certainty.
