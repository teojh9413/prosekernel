# Atlassian content design foundations

## Source
- Author: Atlassian Design System
- URL: https://atlassian.design/foundations/content/

## Why this is good
Atlassian’s content foundations are useful because collaboration tools live or die by shared understanding. The copy has to clarify status, ownership, next action, and consequence without making complex teamwork feel heavier.

## Craft moves
- Connects words to collaboration outcomes.
- Prioritizes user action and shared context.
- Uses system guidance to keep multiple products coherent.
- Treats clarity as a coordination tool.

## Structure map
1. Content role in product experience.
2. Principles for clear and useful language.
3. Guidance for product surfaces and flows.
4. Consistency across ecosystem.

## Reusable pattern
For collaboration products, microcopy should clarify ownership, state, and next action so teams coordinate with less explanation.

## Imitation prompt
Write product copy for a collaboration workflow that names state, owner, next action, and consequence clearly.

## Anti-patterns to avoid
- Using vague collaboration buzzwords.
- Hiding ownership or state behind friendly language.
- Writing labels that make sense only to the product team.
