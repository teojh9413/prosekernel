# Shape Up — Six-Week Cycles

## Source
- Author: Basecamp
- URL: https://basecamp.com/shapeup/2.1-chapter-08
- Rights note: Metadata and original analysis only. Link to the source instead of copying it wholesale.

## Why this is good
Shape Up is valuable as internal operating writing because it converts planning philosophy into constraints teams can follow. The prose names the cycle, the appetite, and the boundary conditions that prevent work from expanding indefinitely.

## Craft moves
- Defines a timebox as an operating constraint.
- Uses appetite to frame tradeoffs before execution.
- Separates shaping from building.
- Treats scope as variable and time as fixed.
- Gives teams language for saying no.

## Structure map
1. Name the operating cycle.
2. Define the appetite or time budget.
3. Clarify what is shaped before work begins.
4. State what changes when scope exceeds appetite.
5. Explain how teams decide done.

## Excerpt or summary
Summary: Shape Up’s cycle guidance is a model for internal planning docs that turn strategy into execution constraints.

## Reusable pattern
For a planning operating doc: “Cycle: [duration]. Appetite: [budget]. Shaped work includes [boundaries]. If scope grows, cut [variable], not [constraint]. Done means [definition].”

## Imitation prompt
Write an internal planning doc using Shape Up discipline. Make appetite, cycle length, scope boundaries, and tradeoffs explicit.

## Anti-patterns to avoid
- Writing planning philosophy without execution rules.
- Treating estimates as commitments while letting scope float invisibly.
- Leaving teams without language for cutting scope.
