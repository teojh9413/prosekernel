# Google SRE Book — Postmortem Culture

## Source
- Author: Google SRE
- URL: https://sre.google/sre-book/postmortem-culture/
- Rights note: Open-license or public documentation source; ProseKernel stores analysis and source link.

## Why this is good
The chapter makes an internal cultural norm operational. “Blameless” is not a slogan; it is a documentation practice that captures facts, contributing factors, remediation, and learning without making people hide information.

## Craft moves
- Defines the purpose of postmortems as learning.
- Makes blamelessness practical rather than sentimental.
- Separates human action from system conditions.
- Turns incidents into tracked remediation.
- Creates a repeatable format for organizational memory.

## Structure map
1. State incident and impact.
2. Document timeline.
3. Identify contributing factors without blame.
4. List remediation and owners.
5. Share lessons and follow-up.

## Excerpt or summary
Summary: Google SRE postmortem culture shows internal docs as organizational learning infrastructure: transparent facts, no scapegoating, clear remediation.

## Reusable pattern
For a blameless postmortem: “Impact: [impact]. Timeline: [events]. Contributing factors: [system/process]. What went well/poorly: [learning]. Actions: [owner/date].”

## Imitation prompt
Write a blameless internal postmortem. Focus on facts, systems, learning, and owned follow-up; avoid blame-shifting or vague lessons.

## Anti-patterns to avoid
- Calling it blameless while implying personal fault.
- Ending with lessons but no owners.
- Hiding uncomfortable facts to protect morale.
