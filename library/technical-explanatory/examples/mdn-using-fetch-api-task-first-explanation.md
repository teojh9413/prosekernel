## Why this is good
MDN’s Fetch guide works because it is task-first. Rather than worshiping the API surface, it organizes around common developer intentions: make a request, send data, process a response, handle details. Examples carry the explanation.

## Craft moves
- Organizes around jobs developers recognize.
- Provides small examples before exhaustive reference.
- Names practical variations.
- Keeps conceptual explanation close to usage.

## Structure map
1. What the API does.
2. Basic usage.
3. Common task variants.
4. Response handling.
5. Caveats and links to reference.

## Reusable pattern
For API docs, lead with common jobs and minimal examples before exhaustive parameters.

## Imitation prompt
Write API documentation that starts with common developer tasks, then shows minimal examples and caveats.

## Anti-patterns to avoid
- Dumping method signatures before use cases.
- Separating every example from the concept it proves.
- Making the reader assemble the common path from reference fragments.
