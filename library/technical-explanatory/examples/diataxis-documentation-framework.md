# Diátaxis Documentation Framework

## Source
- Author: Daniele Procida / Diátaxis
- URL: https://diataxis.fr/
- Rights note: Metadata and analysis only.

## Why this is good
Diátaxis is powerful because it separates four user needs that documentation often mixes: learning, achieving, understanding, and looking up. That separation prevents confusing docs.

## Craft moves
- Defines categories by user need, not writer convenience.
- Gives each category a clear job.
- Prevents tutorials, how-to guides, reference, and explanation from collapsing together.
- Makes documentation architecture teachable.

## Structure map
1. Identify user need.
2. Choose doc type.
3. Write to that mode only.
4. Link to other modes instead of mixing them.

## Excerpt or summary
Summary: documentation should be organized into tutorials, how-to guides, reference, and explanation because each serves a different user need.

## Reusable pattern
Before writing docs, ask: is the reader trying to learn, achieve, understand, or look up?

## Imitation prompt
Classify this documentation request using Diátaxis, then draft the correct doc type without mixing modes.

## Anti-patterns to avoid
- Turning every doc into a tutorial.
- Hiding reference details inside narrative explanation.
