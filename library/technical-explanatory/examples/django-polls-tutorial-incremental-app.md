## Why this is good
The Django polls tutorial works because it avoids abstract tours of the framework. It gives the reader a small application with visible progress, then introduces concepts as needed. The project is small enough to finish and real enough to create mental hooks.

## Craft moves
- Anchors learning in a concrete application.
- Introduces framework pieces only when the project needs them.
- Creates visible milestones.
- Lets repetition build confidence.

## Structure map
1. Create project.
2. Make smallest working piece.
3. Introduce model/view/routing as needed.
4. Iterate toward a complete app.

## Reusable pattern
Teach a framework by building the smallest realistic artifact and introducing abstractions only at the moment of use.

## Imitation prompt
Write a tutorial outline that teaches this framework through one small app with visible milestones and no abstract tour upfront.

## Anti-patterns to avoid
- Cataloging every concept before the reader builds anything.
- Choosing a toy so trivial it teaches no real workflow.
- Skipping verification steps.
