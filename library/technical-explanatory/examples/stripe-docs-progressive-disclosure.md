# Stripe Docs: Progressive Disclosure

## Source
- Author: Stripe
- URL: https://docs.stripe.com/
- Rights note: Link and analysis only.

## Why this is good
Stripe’s documentation is strong because it gives readers control. It combines conceptual hierarchy, copy-pasteable examples, and progressive disclosure so the reader can move from “what is this?” to “how do I implement it?”

## Craft moves
- Starts with the reader’s task.
- Separates concept from implementation.
- Uses examples as proof of clarity.
- Reveals complexity in layers.

## Structure map
1. What you can accomplish.
2. Prerequisites.
3. Minimal working path.
4. Code/example.
5. Edge cases and deeper options.

## Excerpt or summary
Humanprint takeaway: technical writing should reduce cognitive load, not display expertise.

## Reusable pattern
For explainers: task → concept → minimal path → example → edge cases.

## Imitation prompt
Write this technical explanation in a Stripe-docs style: task-first, layered, example-driven, and explicit about prerequisites.

## Anti-patterns to avoid
- Explaining every edge case before the happy path.
- Hiding the concrete example behind conceptual intro.
