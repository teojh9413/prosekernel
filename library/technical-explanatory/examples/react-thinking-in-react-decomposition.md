# React Thinking in React decomposition

## Source
- Author: React Docs Contributors
- URL: https://react.dev/learn/thinking-in-react
- Rights note: Open-license source; Humanprint stores original analysis and uses the source URL for provenance.

## Why this is good
Thinking in React works because it teaches a way to see, not just an API. The guide turns an abstract UI architecture into a repeatable decomposition routine: break the interface down, identify state, decide ownership, then assemble behavior.

## Craft moves
- Uses one concrete interface as the teaching spine.
- Turns expert intuition into numbered decisions.
- Separates visual decomposition from state reasoning.
- Makes the method reusable beyond the example.

## Structure map
1. Introduce a concrete UI target.
2. Break the UI into component parts.
3. Identify minimal state.
4. Choose where state lives.
5. Add interaction in controlled steps.

## Reusable pattern
Teach a design method through one worked example where each step exposes a decision the reader can reuse.

## Imitation prompt
Write a tutorial that teaches a framework mindset through one concrete example, naming each decision and why it comes next.

## Anti-patterns to avoid
- Explaining framework concepts without a worked artifact.
- Mixing all decisions into one large implementation step.
- Treating the example as copy-paste code instead of a mental model.
