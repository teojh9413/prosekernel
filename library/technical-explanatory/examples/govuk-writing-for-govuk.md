# GOV.UK — Writing for GOV.UK

## Source
- Author: Government Digital Service
- URL: https://www.gov.uk/guidance/content-design/writing-for-gov-uk
- Rights note: Metadata and analysis only.

## Why this is good
GOV.UK writing is excellent because it treats clarity as service design. The prose is not trying to impress. It is trying to help a real person complete a real task under real constraints.

## Craft moves
- Starts with user need.
- Uses plain English without condescension.
- Prefers verbs and tasks over institutional language.
- Removes anything that does not help the user act.

## Structure map
1. User need.
2. Task or decision.
3. Plain instruction.
4. Eligibility/edge cases.
5. Next action.

## Excerpt or summary
Summary: write for how people actually read and use government services online; make content accurate, current, plain, and task-focused.

## Reusable pattern
For service writing: “You can [do thing] if [condition]. You need [items]. It takes [time/cost]. Start by [action].”

## Imitation prompt
Rewrite this process in GOV.UK style: user need first, plain verbs, short sentences, no institutional fog, clear next action.

## Anti-patterns to avoid
- Mistaking plain language for childish language.
- Explaining the institution’s internal logic before the user’s task.
