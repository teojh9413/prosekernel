# Superhuman inbox zero premium promise

## Source
- Author: Superhuman
- URL: https://superhuman.com/
- Rights note: Metadata and original craft analysis only; do not mirror full source text.

## Why this is good
Superhuman is useful as copy because it sells a felt workflow state, not a pile of email features. The reader already knows inbox pain, so the page can frame speed, focus, and status as the proof that the product changes a daily habit.

## Craft moves
- Names the desired state before the feature set.
- Treats speed as emotional relief, not only performance.
- Uses premium positioning to filter for buyers who value time highly.
- Lets product capability support a concrete before/after.

## Structure map
1. Reader pain: inbox drag and attention loss.
2. Desired state: faster, calmer, more controlled email.
3. Capability proof: speed, shortcuts, triage, workflow fit.
4. Premium conversion path for high-intent users.

## Reusable pattern
For premium workflow products, sell the expensive daily friction first, then show why the product is worth disproportionate attention and price.

## Imitation prompt
Write premium SaaS copy for a painful daily workflow. Lead with the desired state, prove the workflow change, and make the high price feel tied to time saved.

## Anti-patterns to avoid
- Opening with generic productivity claims.
- Listing features before naming the daily pain.
- Using premium tone without concrete workflow proof.
