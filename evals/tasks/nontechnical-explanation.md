# Eval Task — Technical Explanation for Non-Technical Users

## Task
Explain API rate limits to a non-technical founder deciding whether their agent product can launch this week.

## Reader
A founder who needs decision clarity, not implementation detail.

## Required pattern coverage
- `PATTERN_EXPLAIN_001`
- `PATTERN_STRATEGY_001`

## Must demonstrate
- Progressive disclosure: practical answer first, mechanics second, edge cases last.
- A concrete example with numbers.
- Decision implication and next action.

## Failure modes
- Dictionary definition with no decision help.
- Too much protocol detail too early.
- No example calculation.
