# Example Quality Rubric

Use this rubric before adding a Humanprint example.

## Accept if

The example teaches a transferable craft move and has clear source provenance.

Score each dimension 1-5:

1. **Specificity** — Does it use concrete details, names, numbers, scenes, or precise claims?
2. **Structure** — Does it reveal an argument, persuasion, explanation, or narrative sequence agents can reuse?
3. **Reader fit** — Does it match the reader's awareness, context, and next action?
4. **Proof / credibility** — Does it earn trust through evidence, observation, demonstration, or honest constraint?
5. **Memorability** — Does it compress meaning in a way that sticks without becoming gimmicky?
6. **Transferability** — Can the move be applied to other writing tasks without copying wording?
7. **Rights safety** — Can we store metadata/analysis safely without mirroring protected text?

## Reject if

- It is popular but teaches no reusable move.
- It is only included because the author is famous.
- It requires copying too much copyrighted text to be useful.
- It is clever but not useful.
- It is generic advice disguised as an example.
- It encourages AI to imitate voice instead of transfer structure.

## Quality score guide

- `10`: canonical example; teaches several durable moves.
- `8-9`: strong example; teaches at least one move unusually well.
- `6-7`: acceptable but replaceable; use only if category needs coverage.
- `<6`: do not include.

## Public repo standard

Humanprint should be high-signal. A smaller rights-safe corpus with sharp annotations is better than a large messy swipe file.
